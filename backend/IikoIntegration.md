## Интеграция с iiko (самовывоз)

### Переменные окружения

В `.env` или `backend/.env` задайте:

- `IIKO_BASE_URL` — базовый URL демо‑сервера iikoRMS, по умолчанию `https://990-418-833.iiko.it/resto/`.
- `IIKO_LOGIN` — логин для доступа к API (например, `user`).
- `IIKO_PASSWORD` — пароль для доступа к API (например, `user#test`).
- `IIKO_ORGANIZATION_ID` — идентификатор организации в iiko (может быть получен из BackOffice/документации API).
- `IIKO_TERMINAL_ID` — идентификатор кассы/терминала самовывоза.

### HTTP‑эндпоинт для создания заказа самовывоза

- Метод: `POST`
- URL: `/api/v1/iiko/orders/pickup`
- Тело запроса (JSON):

```json
{
  "customer": {
    "name": "Иван",
    "phone": "+79990000000"
  },
  "items": [
    {
      "productId": "ID_ТОВАРА_ИЗ_СПРАВОЧНИКА_IIKO",
      "quantity": 1
    }
  ],
  "comment": "Комментарий к заказу",
  "pickupAt": "2026-03-10T18:30:00+03:00"
}
```

### Примечания

- Эндпоинт создаёт заказ типа **самовывоз** через iikoRMS REST API (delivery order без адреса).
- Ответ содержит статус и «сырое» тело ответа от iiko, что позволяет отлаживать интеграцию и сверять поля с официальной документацией.
